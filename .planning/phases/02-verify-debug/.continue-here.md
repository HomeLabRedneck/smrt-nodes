# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: ZIP-base-directory — ZIP must be built from the MOD DIRECTORY (e.g. `Smrt Thread Manager/`), NOT from `mods-unpacked/`. Using `mods-unpacked/` as base produces entries like `kuuk-SmartThreadManager/manifest.json` which lack the required `mods-unpacked/` prefix — mod loader silently fails.
- [ ] CONSTRAINT: ZIP-path-separator — Always `.NET ZipArchive` with `path.Replace('\', '/')`. Never `Compress-Archive`.
- [ ] CONSTRAINT: update_connections-dispatch — `connection_out_set` signal hook + tick fallback in `_ready()` are the core fix. Never remove them.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Wrong ZIP base directory | Using `mods-unpacked/` as packaging base → entries missing `mods-unpacked/` prefix → mod loader can't find manifest | blocking | Always use parent mod directory as base; verify first 3 entries start with `mods-unpacked/` |
| Compress-Archive packaging | Produces backslash paths → mod loader can't find files | blocking | Always .NET ZipArchive with path.Replace |
| Removing connection_out_set hook | Breaks distribution entirely (transfer stays 0) | blocking | Never remove from _ready() in smart_resource_container.gd |
