# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: ZIP-base-directory — ZIP must be built from MOD DIRECTORY (e.g. `Smrt GPU Manager/`), NOT from `mods-unpacked/`. Wrong base produces entries without `mods-unpacked/` prefix — mod loader silently fails.
- [ ] CONSTRAINT: ZIP-path-separator — Always `.NET ZipArchive` with `Add-Type -AssemblyName System.IO.Compression` + `Add-Type -AssemblyName System.IO.Compression.FileSystem`. Never `Compress-Archive`.
- [ ] CONSTRAINT: update_connections-dispatch — `connection_out_set` signal hook + tick fallback in `_ready()` are the core fix. Never remove them.
- [ ] CONSTRAINT: SGM-role-in-set_containers — SGM role assignment lives in `set_containers()`, NOT `_init()`. provided is empty at init time. Never move it back to _init().
- [ ] CONSTRAINT: STM-unchanged — Do NOT apply SGM consumer classification change to STM. STM works correctly as-is.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Wrong ZIP base directory | Using `mods-unpacked/` as base → entries missing `mods-unpacked/` prefix → mod loader can't find manifest | blocking | Always use parent mod directory as base; verify first entry starts with `mods-unpacked/` |
| Compress-Archive packaging | Produces backslash paths → mod loader can't find files | blocking | Always `Add-Type -AssemblyName System.IO.Compression` + `Add-Type -AssemblyName System.IO.Compression.FileSystem`, use ZipArchive directly |
| Removing connection_out_set hook | Breaks distribution entirely (transfer stays 0) | blocking | Never remove from _ready() in smart_resource_container.gd |
| SGM role in _init | Role set at _init when provided=[] → always STORAGE → demand/graph = ratio | blocking | Role must be in set_containers() where provided is populated |
| Applying SGM fix to STM | STM classification is different — STM uses dependent (multi-resource windows). Applying !provided.is_empty() to STM breaks STM | blocking | Only touch SGM stm_window_data.gd |
