# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: ZIP-base-directory — ZIP must be built from MOD DIRECTORY (e.g. `Smrt Thread Manager/`), NOT from `mods-unpacked/`. Wrong base produces entries without `mods-unpacked/` prefix — mod loader silently fails.
- [ ] CONSTRAINT: ZIP-path-separator — Always `.NET ZipArchive` with `path.Replace('\', '/')`. Never `Compress-Archive`.
- [ ] CONSTRAINT: update_connections-dispatch — `connection_out_set` signal hook + tick fallback in `_ready()` are the core fix. Never remove them.
- [ ] CONSTRAINT: count-demand-feedback — Do NOT use `inputs[n].count` as demand proxy for windows without `required`. Creates feedback loop in Graph mode. Use `required` if present, else constant `1.0`.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Wrong ZIP base directory | Using `mods-unpacked/` as base → entries missing `mods-unpacked/` prefix → mod loader can't find manifest | blocking | Always use parent mod directory as base; verify first entry starts with `mods-unpacked/` |
| Compress-Archive packaging | Produces backslash paths → mod loader can't find files | blocking | Always .NET ZipArchive with path.Replace |
| Removing connection_out_set hook | Breaks distribution entirely (transfer stays 0) | blocking | Never remove from _ready() in smart_resource_container.gd |
| count-based demand fallback | inputs[n].count as demand → feedback loop in Graph mode → allocations decay to ~0.25Hz then persist after mode switch | blocking | Use required if available, else 1.0 constant. Removed from stm_window_data.gd get_demand/get_count_demand |
